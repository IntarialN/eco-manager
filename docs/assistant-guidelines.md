# Правила работы для ИИ

1. Перед началом любой задачи:
   - Прочитай `docs/documentation-plan.md` и релевантные разделы по теме.
   - Проверь `docs/completed/` на предмет уже выполненных пунктов, чтобы не дублировать работу.
2. Соблюдай структуру проекта:
   - Разделяй функционал на самостоятельные модули и сервисы.
   - При проектировании учитывай требования к RLS, безопасности и законодательству РФ.
3. Все изменения сопровождай документацией:
   - После завершения этапа добавляй запись в `docs/completed/`.
   - Обновляй индекс (`docs/README.md`), если появляются новые документы или разделы.
4. В коммуникации и коде:
   - Указывай источники требований (документы, НПА, анкеты), если на них ссылаешься.
   - Используй понятные комментарии только там, где логика неочевидна.
5. Перед реализацией функционала уточняй:
   - Нужны ли интеграции (учётная система, внешние сервисы).
   - Какие статусы/ролей задействованы, чтобы не нарушить модель доступа.
6. Тестирование и проверка:
   - Для каждого блока описывай критерии приёмки и список обязательных проверок.
   - Не переходи к следующему этапу, пока текущий не зафиксирован в `docs/completed/`.
