## 3. Открытые вопросы

- Свежий список TODO доступен в `docs/documentation-todo.md`.
- Перед стартом реализации необходимо приоритизировать и закрыть критичные пункты (req_11, API Bubble, НПА, безопасность).
- Прогресс по закрытию фиксируем отдельными completed-записями с ссылкой на обновлённые документы.

## 4. Рекомендации перед стартом разработки

1. Провести с заказчиком сессию уточнений по открытым вопросам (`documentation-todo.md`).
2. Утвердить технологический стек и обновить `docs/architecture/code-structure.md`/`overview.md`.
3. Подготовить визуальные диаграммы (архитектура, ERD, последовательности) для onboarding команды.
4. Настроить репозиторий согласно структуре и запустить базовый CI (lint + тесты).

## 5. Статус готовности

- Основная документация подготовлена и покрывает все блоки MVP.
- Открытые вопросы зафиксированы и не мешают началу прототипирования, но требуют решения до релиза.
- Документирование процессов ведения знаний и журнала завершено — последующие сессии могут продолжить работу без потери контекста.
