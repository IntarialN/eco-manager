# ADR 0003 — Переход на Yii2 монолит

- Дата: 2025-11-06
- Статус: Принято
- Связанные решения: ADR 0001 (NestJS/React), ADR 0002 (инфраструктура)

## Контекст

Исходно предполагалось использовать стек Nx + NestJS + React (см. ADR 0001). Заказчик уточнил обязательное требование — backend должен быть реализован на Yii2. Для MVP достаточно веб-интерфейса с пятью основными блоками (карта требований, артефакты, календарь, риски, договоры).

## Решение

- Отказаться от стека NestJS/React для MVP в пользу Yii2 (basic template).
- Реализовать личный кабинет как монолитное приложение с вкладками, основанными на ActiveRecord-моделях.
- Использовать SQLite как демо-БД (лёгкая миграция на PostgreSQL/MySQL при внедрении).
- Сохранить текущие архитектурные принципы (чёткое разделение сущностей и блоков, документация, миграции, completed-лог).

## Последствия

- Индивидуальные сервисы и фронтенд SPA из ADR 0001 не реализуются в текущем прототипе.
- Необходимо обновить документацию (code-structure, overview, runbook).
- Инфраструктурный ADR 0002 остаётся актуальным для будущего production, но MVP разворачивается стандартными средствами Yii2.
- Следующие шаги: реализовать RBAC, загрузку документов и интеграцию с внешними системами поверх Yii2, либо выделить сервисы позднее.
