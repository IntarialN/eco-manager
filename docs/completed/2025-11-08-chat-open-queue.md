# Возврат чатов в «Открытые» при новых сообщениях

**Контекст**
- Требование: новые сообщения должны снова попадать в очередь «Открытые», чтобы их могли взять другие операторы.

**Что сделано**
- `ChatService::postMessage` теперь сбрасывает `assigned_user_id`/`assigned_seen_at`, когда сообщение отправляет клиент. Чат меняет статус на `open`, поэтому появляется во вкладке «Открытые».
- `ChatSession` получил поле `assigned_seen_at` (см. миграцию `m230000_000011_add_chat_seen_fields.php`), логика обновления уже задействована на UI.
- Документ `docs/features/chat-support.md` дополнен описанием поведения.

**Следующее**
- Подключить realtime (Centrifugo) и события возврата в очередь, чтобы UI обновлялся без polling.
