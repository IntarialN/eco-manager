# Журнал завершённых задач

Каждое выполненное действие фиксируем отдельным файлом в этой папке. Журнал служит источником истины для отслеживания прогресса и контрольных точек по `docs/documentation-plan.md`.

## Формат записи
- Имя файла: `YYYY-MM-DD-краткое-название.md`.
- Содержимое:
  - **Контекст** — на какой пункт из `docs/documentation-plan.md` или другого документа ссылаемся.
  - **Что сделано** — краткое описание результата.
  - **Артефакты** — ссылки на обновлённые файлы или коммиты.
  - **Следующие шаги** — если требуется.

## Процесс ведения
1. Перед началом работы проверяем, есть ли в журнале запись по соответствующему этапу. Если нет — готовим новую.
2. После выполнения задачи добавляем запись в тот же день, даже если результат промежуточный.
3. Если пункт требует дополнительных уточнений или имеются открытые TODO — фиксируем их в блоке «Следующие шаги».
4. При повторном обращении к задаче создаём новую запись с актуальной датой, не переписываем историю.

## Ежедневный аудит
- Убедиться, что каждый completed-файл ссылается на конкретные артефакты.
- Проверить отсутствие пропущенных этапов плана (`docs/documentation-plan.md`).
- Пройтись по `rg "TODO" docs` и обновить таблицу открытых вопросов (`docs/documentation-todo.md`, см. ниже).
- В конце недели готовить сводную запись (например, `YYYY-MM-DD-weekly-summary.md`) с кратким статусом и рисками.

## Связанные документы
- `docs/documentation-todo.md` — сводная таблица открытых задач и уточнений (обновляем при каждом аудите).

## Правила
- Если задача частично завершена, фиксируем межрезультаты и явно указываем, что осталось.
- При изменении записи не удаляем историю — добавляем новый файл с актуальной датой.
