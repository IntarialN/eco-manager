# Шаблон анкеты клиента

Цель анкеты — собрать исходные данные для автоматического построения карты требований, настройки календаря и управления договорами. Перед заполнением менеджер проверяет актуальность данных у клиента и при необходимости запрашивает подтверждающие документы.

## 1. Общая информация
- Полное юридическое наименование
- ИНН / ОГРН
- Контактное лицо (ФИО, должность, телефон, e-mail)
- Группа компаний / холдинг (если применимо)
- Роли пользователей, которым требуется доступ к кабинету

## 2. Объекты и категория по НВОС
- Перечень объектов (площадки, филиалы) с адресами
- Категория НВОС по каждому объекту (I, II, III, IV)
- Основание присвоения категории (решение, номер и дата)
- Наличие стационарных источников выбросов
- Наличие радиоактивных, высокотоксичных веществ (класс I–II)

## 3. Производственная деятельность
- Основной вид деятельности (ОКВЭД)
- Источники выбросов / виды отходов
- Объём выбросов за прошлый год (т/год)
- Образование отходов (кг/год) по классам опасности
- Наличие побочной продукции животноводства

## 4. Водопользование
- Наличие артезианских/буровых скважин (номер лицензии, срок)
- Забор воды из поверхностных источников (рек, озёр) — реквизиты решения/договора
- Обратные сбросы (если есть) и наличие разрешений

## 5. Документы и требования
- Существующие журналы учёта (отходы, источники выбросов)
- Наличие паспортов отходов, НООЛР, НДВ, ППЭК, ДВОС, КЭР
- Статус программ производственного контроля
- Даты отчётности и последние поданные формы (2-ТП, ПЭК, др.)

## 6. Ответственные лица и обучение
- Перечень ответственных за экологию по объектам
- Наличие действующих удостоверений об обучении и даты истечения
- Необходимость планового обучения/переобучения

## 7. Договоры и расчёты
- Действующие договоры с ООО «ГлавПриродБюро» (номер, дата, предмет, статус)
- Статус счетов и актов (оплачен, в работе, просрочен)
- Потребность в интеграции с учётной системой (Бабл) и реквизиты для API

## 8. Дополнительные сведения
- История проверок и штрафов (за последние 3 года)
- Планируемые изменения в деятельности (расширение, модернизация)
- Требования к отчётности/аналитике со стороны клиента
- Каналы коммуникации (чат-бот, обратный звонок, SLA реакции)

## 9. Артефакты и вложения
- Список загружаемых документов (формат, дата, ответственный)
- Права доступа на документы (какие роли видят/редактируют)
- Необходимость аудита при загрузке (да/нет, условия)

- [x] Обозначены данные для интеграции с договорами/счетами.
- [x] Указаны поля для контроля обучения ответственных.
- [x] Предусмотрено хранение артефактов и права доступа.
